# Demo Practice

Spring Boot API practice project that sends user prompts to a local Ollama model through Spring AI.

## What It Does

The application exposes one API:

```text
GET /prompt/result?source={source}&prompt={prompt}
```

The API returns:

- `source`: the caller-provided source value
- `agentName`: the configured Ollama chat model
- `result`: the response generated by the local Ollama model

## Tech Stack

- Java 22
- Spring Boot 3.5.14
- Spring AI Ollama
- Maven
- Docker Compose
- JUnit 5, Mockito, MockMvc
- GitHub Actions on Ubuntu

## Prerequisites

- Java 22
- Docker Desktop or Docker Engine with Docker Compose
- Maven, or use the included Maven wrapper

On Windows, use `mvnw.cmd`. On Linux/macOS/Git Bash, use `./mvnw`.

## Local Ollama

Start the local Ollama stack:

```bash
docker compose up -d
```

The compose stack starts:

- `ollama`: local Ollama server on `http://localhost:11434`
- `ollama-model`: one-shot service that pulls `llama3.2:1b`

Check that Ollama is available:

```bash
docker compose ps
```

You can also verify the model list:

```bash
docker compose exec ollama ollama list
```

## Run The App

PowerShell:

```powershell
$env:OLLAMA_BASE_URL="http://localhost:11434"
$env:OLLAMA_CHAT_MODEL="llama3.2:1b"
.\mvnw.cmd spring-boot:run
```

Bash:

```bash
OLLAMA_BASE_URL=http://localhost:11434 OLLAMA_CHAT_MODEL=llama3.2:1b ./mvnw spring-boot:run
```

Default configuration is already set in `src/main/resources/application.properties`:

```properties
spring.ai.ollama.base-url=${OLLAMA_BASE_URL:http://localhost:11434}
spring.ai.ollama.chat.model=${OLLAMA_CHAT_MODEL:llama3.2:1b}
```

## API Example

Request:

```bash
curl -iSs -X GET "http://localhost:8080/prompt/result?source=dummy&prompt=who_am_i"
```

Response shape:

```json
{
  "source": "dummy",
  "agentName": "llama3.2:1b",
  "result": "<response from local Ollama>"
}
```

The `result` value is generated by Ollama, so it can vary between calls.


It runs on:

- pushes to `main`
- pull requests targeting `main`
- manual workflow dispatch

The workflow uses Ubuntu, Java 22, Maven dependency caching, and runs:

```bash
./mvnw test
```

## Configuration

| Property | Environment Variable | Default |
| --- | --- | --- |
| `spring.ai.ollama.base-url` | `OLLAMA_BASE_URL` | `http://localhost:11434` |
| `spring.ai.ollama.chat.model` | `OLLAMA_CHAT_MODEL` | `llama3.2:1b` |

To use a different model:

```bash
docker compose exec ollama ollama pull mistral
```

Then run the app with:

```bash
OLLAMA_CHAT_MODEL=mistral ./mvnw spring-boot:run
```

## Useful Commands

Package the app:

```bash
./mvnw clean package
```

Stop the local stack:

```bash
docker compose down
```

Stop the stack and remove the persisted Ollama model volume:

```bash
docker compose down -v
```

## Troubleshooting

If the API fails to respond from Ollama, check:

- Docker containers are running with `docker compose ps`
- the model is available with `docker compose exec ollama ollama list`
- `OLLAMA_BASE_URL` points to `http://localhost:11434` when the Spring app runs on the host machine
- the first prompt can be slower because Ollama has to load the model
